# Database Schema Documentation

Complete reference for StreamIt database models and relationships.

## Overview

The database uses **PostgreSQL** with **Prisma ORM**. The schema is organized into logical modules:
- Authentication & Users
- Creator Applications
- Live Streaming
- Social Features
- Content Management
- Payments & Monetization
- Discount System

## Core Models

### User
The central entity representing all users (viewers and creators).

```prisma
model User {
  id            String   @id
  name          String
  age           Int?
  email         String   @unique
  phone         String?
  username      String   @unique
  bio           String?  @db.Text
  emailVerified Boolean  @default(false)
  image         String?
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}
```

**Fields**:
- `id` - Unique identifier (generated by Better Auth)
- `email` - Unique email address
- `username` - Unique username for profile URL
- `bio` - User biography (text field)
- `emailVerified` - Whether email is verified
- `image` - Profile picture URL

**Relationships**:
- `sessions` - Active sessions
- `accounts` - Connected accounts (email, social)
- `creatorApplication` - Creator onboarding application
- `stream` - Live stream configuration (creators only)
- `following/followedBy` - Follow relationships
- `blocking/blockedBy` - Block relationships
- `posts` - User's content posts
- `likes/comments` - Engagement on posts
- `coinWallet` - Virtual currency wallet
- `coinPurchases` - Purchase history
- `giftsSent/giftsReceived` - Gift transactions
- `ownedDiscountCodes` - User's reward discount codes

## Authentication Models

### Session
Manages user authentication sessions.

```prisma
model Session {
  id        String   @id
  expiresAt DateTime
  token     String   @unique
  username  String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  ipAddress String?
  userAgent String?
  userId    String
  user      User     @relation(...)
}
```

**Key Points**:
- Created by Better Auth
- Supports cookie and bearer token authentication
- Tracks IP address and user agent
- Automatic expiration handling

---

### Account
Stores authentication provider accounts.

```prisma
model Account {
  id                    String    @id
  accountId             String
  providerId            String    // "email", "google", etc.
  userId                String
  user                  User      @relation(...)
  accessToken           String?
  refreshToken          String?
  idToken               String?
  accessTokenExpiresAt  DateTime?
  refreshTokenExpiresAt DateTime?
  scope                 String?
  password              String?   // Hashed
  createdAt             DateTime  @default(now())
  updatedAt             DateTime  @updatedAt
}
```

**Key Points**:
- Supports multiple providers per user
- Password is hashed using Better Auth
- Stores OAuth tokens for social login

---

### Verification
Temporary storage for email OTPs and verification codes.

```prisma
model Verification {
  id         String   @id
  identifier String   // Email or phone
  value      String   // OTP code
  expiresAt  DateTime
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt
}
```

**Key Points**:
- Auto-expires after configured time
- Used for email verification, password reset
- Managed by Better Auth

## Creator Application Models

### CreatorApplication
Main application for becoming a creator.

```prisma
model CreatorApplication {
  id              String            @id @default(cuid())
  userId          String            @unique
  user            User              @relation(...)
  status          ApplicationStatus @default(DRAFT)
  submittedAt     DateTime?
  reviewedAt      DateTime?
  reviewedBy      String?
  rejectionReason String?
  identity        IdentityVerification?
  financial       FinancialDetails?
  profile         CreatorProfile?
  createdAt       DateTime          @default(now())
  updatedAt       DateTime          @updatedAt
}
```

**Status Flow**:
1. `DRAFT` - User started application
2. `PENDING` - Submitted for review
3. `UNDER_REVIEW` - Admin is reviewing
4. `APPROVED` - Approved, can stream
5. `REJECTED` - Rejected with reason

**Sub-models**:
- `IdentityVerification` - ID documents and selfie
- `FinancialDetails` - Bank details for payouts
- `CreatorProfile` - Public profile info

## Streaming Models

### Stream
Live stream configuration (one per creator).

```prisma
model Stream {
  id                  String            @id @default(cuid())
  title               String            @db.Text
  description         String?           @db.Text
  thumbnail           String?           @db.Text
  isLive              Boolean           @default(false)
  isChatEnabled       Boolean           @default(true)
  isChatDelayed       Boolean           @default(false)
  isChatFollowersOnly Boolean           @default(false)
  userId              String            @unique
  user                User              @relation(...)
  giftTransactions    GiftTransaction[]
  createdAt           DateTime          @default(now())
  updatedAt           DateTime          @updatedAt
}
```

**Key Fields**:
- `isLive` - Current streaming status
- `isChatEnabled` - Allow chat messages
- `isChatDelayed` - Delay chat by X seconds
- `isChatFollowersOnly` - Only followers can chat

**Indexes**:
- `userId` - Fast creator lookup
- `title` - Search optimization
- `isLive` - Filter live streams

## Social Models

### Follow
User follow relationships.

```prisma
model Follow {
  id          String   @id @default(cuid())
  followerId  String
  followingId String
  follower    User     @relation("Following", ...)
  following   User     @relation("FollowedBy", ...)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  
  @@unique([followerId, followingId])
}
```

**Constraints**:
- Unique pair prevents duplicate follows
- Indexed for performance

---

### Block
User block relationships.

```prisma
model Block {
  id        String   @id @default(cuid())
  blockerId String
  blockedId String
  blocker   User     @relation("Blocking", ...)
  blocked   User     @relation("BlockedBy", ...)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  
  @@unique([blockerId, blockedId])
}
```

**Behavior**:
- Blocked users cannot see blocker's content
- Automatic unfollow on block
- Hide from search/recommendations

## Content Models

### Post
User-generated content posts.

```prisma
model Post {
  id            String      @id @default(cuid())
  content       String?     @db.Text
  type          PostType    @default(TEXT)
  media         PostMedia[]
  likes         Like[]
  comments      Comment[]
  likesCount    Int         @default(0)
  commentsCount Int         @default(0)
  isPublic      Boolean     @default(true)
  allowComments Boolean     @default(true)
  authorId      String
  author        User        @relation(...)
  createdAt     DateTime    @default(now())
  updatedAt     DateTime    @updatedAt
}
```

**Post Types**: `TEXT`, `IMAGE`, `VIDEO`, `MIXED`

**Counts**:
- `likesCount` and `commentsCount` are denormalized for performance
- Updated via triggers or application logic

### Like
Post likes.

```prisma
model Like {
  id        String   @id @default(cuid())
  userId    String
  postId    String
  user      User     @relation(...)
  post      Post     @relation(...)
  createdAt DateTime @default(now())
  
  @@unique([userId, postId])
}
```

**Constraints**:
- One like per user per post
- Indexed for fast lookup

---

### Comment
Comments on posts (supports nesting).

```prisma
model Comment {
  id         String        @id @default(cuid())
  content    String        @db.Text
  userId     String
  postId     String
  parentId   String?
  user       User          @relation(...)
  post       Post          @relation(...)
  parent     Comment?      @relation("CommentReplies", ...)
  replies    Comment[]     @relation("CommentReplies")
  likes      CommentLike[]
  likesCount Int           @default(0)
  createdAt  DateTime      @default(now())
  updatedAt  DateTime      @updatedAt
}
```

**Nested Comments**:
- `parentId` is null for top-level comments
- `parentId` references another comment for replies
- Supports unlimited nesting depth

### CoinPurchase
Record of coin purchases.

```prisma
model CoinPurchase {
  id                 String              @id @default(cuid())
  userId             String
  user               User                @relation(...)
  packageId          String
  package            CoinPackage         @relation(...)
  coins              Int
  bonusCoins         Int                 @default(0)
  totalCoins         Int
  amount             Int
  currency           String              @default("INR")
  discountCodeId     String?
  discountBonusCoins Int                 @default(0)
  discountRedemption DiscountRedemption?
  paymentGateway     String?             @default("dodo")
  transactionId      String?             @unique
  orderId            String              @unique
  status             PurchaseStatus      @default(PENDING)
  failureReason      String?
  paymentData        Json?
  createdAt          DateTime            @default(now())
  updatedAt          DateTime            @updatedAt
}
```

**Status Flow**:
- `PENDING` - Checkout created
- `COMPLETED` - Payment successful, coins credited
- `FAILED` - Payment failed
- `REFUNDED` - Refund issued

**Total Coins Calculation**:
```
totalCoins = coins + bonusCoins + discountBonusCoins
```

## Discount System Models

### DiscountCode
Promotional and reward discount codes.

```prisma
model DiscountCode {
  id                 String               @id @default(cuid())
  code               String               @unique
  discountType       DiscountType         // PERCENTAGE, FIXED
  discountValue      Int
  codeType           CodeType             // PROMOTIONAL, REWARD
  createdBy          String?              // Admin ID
  ownerId            String?              // User ID (for rewards)
  owner              User?                @relation(...)
  maxRedemptions     Int?
  currentRedemptions Int                  @default(0)
  isOneTimeUse       Boolean              @default(true)
  minPurchaseAmount  Int?
  expiresAt          DateTime?
  isActive           Boolean              @default(true)
  description        String?
  redemptions        DiscountRedemption[]
  createdAt          DateTime             @default(now())
  updatedAt          DateTime             @updatedAt
}
```

**Types**:
- **Promotional**: Created by admins, public, reusable
  - Example: `WELCOME10` - 10% off for new users
- **Reward**: Auto-generated for users after purchase, one-time use
  - Example: `USER_ABC_REWARD123` - 5% bonus coins

**Discount Calculation**:
- `PERCENTAGE`: `bonusCoins = (packageCoins * discountValue) / 100`
- `FIXED`: `bonusCoins = discountValue`

---

### DiscountRedemption
Track discount code usage.

```prisma
model DiscountRedemption {

  discountCodeId    String
  discountCode      DiscountCode @relation(...)

  user              User         @relation(...)
  purchaseId        String       @unique
  purchase          CoinPurchase @relation(...)
  bonusCoinsAwarded Int
  createdAt         DateTime     @default(now())
}
```

**Constraints**:
- One redemption per purchase
- Tracks bonus coins awarded

## Database Queries

### Common Patterns

**Get user with relations**:
```typescript
const user = await prisma.user.findUnique({
  where: { id: userId },
  include: {
    stream: true,
    coinWallet: true,
    creatorApplication: {
      include: {
        identity: true,
        financial: true,
        profile: true,
      },
    },
  },
});
```

**Get live streams**:
```typescript
const liveStreams = await prisma.stream.findMany({
  where: { isLive: true },
  include: {
    user: {
      select: {
        id: true,
        username: true,
        name: true,
        image: true,
      },
    },
  },
  orderBy: { updatedAt: 'desc' },
});
```

**Get feed with pagination**:
```typescript
const posts = await prisma.post.findMany({
  where: { isPublic: true },
  include: {
    author: true,
    media: true,
    _count: {
      select: {
        likes: true,
        comments: true,
      },
    },
  },
  skip: (page - 1) * limit,
  take: limit,
  orderBy: { createdAt: 'desc' },
});
```
